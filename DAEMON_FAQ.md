# 守护者进程模式 - 常见问题解答

## ❓ 每天都是获取最新的公告吗？

**答案：是的，但有两种模式可选择：**

### 🔄 增量下载模式（推荐）
- **默认启用**：只获取最近几天的公告（默认7天）
- **高效快速**：避免重复下载已有文件
- **适合场景**：日常定时下载

```yaml
daemon:
  tasks:
    incremental_mode: true          # 启用增量下载
    incremental_days: 7             # 获取最近7天的公告
```

### 📅 完整范围模式
- **使用任务配置的日期范围**：从start_date到end_date的所有公告
- **完整下载**：确保不遗漏任何历史数据
- **适合场景**：首次运行或需要完整历史数据

```yaml
daemon:
  tasks:
    incremental_mode: false         # 禁用增量下载
```

## 🕐 具体的执行时间

### 默认调度时间
- **上午 09:00**：获取前一天晚上到今天上午的新公告
- **下午 18:00**：获取今天下午的新公告

### 自定义调度时间
```yaml
daemon:
  schedule:
    times: ["09:30", "12:00", "18:30"]  # 一天三次
```

## 📊 实际下载内容

### 增量模式下（推荐设置）
```yaml
incremental_days: 3  # 最近3天
```
- **每天09:00执行**：下载最近3天的公告
- **每天18:00执行**：下载最近3天的公告
- **智能去重**：已存在的文件会被跳过

### 为什么选择3天？
1. **覆盖周末**：确保周末发布的公告不会遗漏
2. **网络容错**：如果某次执行失败，下次还能补上
3. **效率平衡**：既保证完整性又不会下载太多重复内容

## 🎯 推荐配置

### 日常使用（最佳实践）
```yaml
daemon:
  enabled: true
  schedule:
    times: ["09:30", "18:30"]       # 避开交易开盘时间
  tasks:
    run_all_enabled: true
    incremental_mode: true
    incremental_days: 3             # 最近3天，确保不遗漏
    run_on_startup: false
```

### 首次运行
```yaml
daemon:
  tasks:
    incremental_mode: false         # 先下载完整历史数据
```
运行一段时间后，再改为：
```yaml
daemon:
  tasks:
    incremental_mode: true          # 切换到增量模式
    incremental_days: 3
```

## 📈 性能对比

### 增量模式 vs 完整模式

| 模式 | 下载范围 | 执行时间 | 网络负载 | 推荐场景 |
|------|----------|----------|----------|----------|
| 增量模式 | 最近N天 | 快速 | 低 | 日常定时下载 |
| 完整模式 | 完整日期范围 | 较慢 | 高 | 首次运行/历史数据 |

### 实际例子
假设任务配置：
```yaml
- name: "腾讯控股"
  stock_code: "00700"
  start_date: "2024-01-01"  # 从年初开始
  end_date: "today"
```

- **完整模式**：每次都下载2024-01-01到今天的所有公告
- **增量模式**：只下载最近3天的公告（如2024-12-13到2024-12-15）

## 🔧 如何调整

### 修改增量天数
```yaml
daemon:
  tasks:
    incremental_days: 1   # 只获取今天的（适合一天多次执行）
    incremental_days: 7   # 获取一周的（适合每周执行）
    incremental_days: 30  # 获取一个月的（适合每月执行）
```

### 临时禁用增量模式
```bash
# 在配置文件中设置
incremental_mode: false

# 然后重启守护者进程
python main.py --daemon-restart
```

## 💡 最佳实践建议

1. **首次使用**：先用完整模式下载历史数据
2. **日常运行**：使用增量模式，设置3-7天
3. **监控日志**：定期检查是否有遗漏
4. **定期全量**：每月或每季度运行一次完整模式
5. **错峰执行**：避开交易时间和网络高峰期
