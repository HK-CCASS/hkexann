# 手动历史公告补充脚本使用说明

## 📖 功能概述

`manual_historical_backfill.py` 是一个专用的手动历史公告补充脚本，复用现有系统组件，为指定股票手动补充历史公告数据。

### 🎯 主要特性

- **🔄 组件复用**: 完全复用现有的下载器、向量化器、过滤器等组件
- **📅 灵活日期**: 支持指定天数或具体日期范围
- **📊 批量处理**: 支持单个股票或批量股票处理
- **🔬 智能过滤**: 可选的公告过滤功能
- **📈 详细统计**: 完整的处理进度和效率统计
- **🎭 试运行**: 支持试运行模式，预览处理内容

## 🚀 快速开始

### 基本用法

```bash
# 为腾讯(00700)补充最近7天的历史公告
python manual_historical_backfill.py -s 00700 -d 7

# 为多只股票补充最近30天的历史公告
python manual_historical_backfill.py -s "00700,00939,01398" -d 30

# 指定具体日期范围
python manual_historical_backfill.py -s 00700 --date-range 2025-09-10 2025-09-17
```

### 从文件读取股票列表

```bash
# 使用示例股票文件
python manual_historical_backfill.py -s example_stocks.txt -d 7

# 使用自定义股票文件
python manual_historical_backfill.py -s my_stocks.txt -d 30
```

## 📋 命令行参数详解

### 必需参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `-s, --stocks` | 股票代码列表 | `00700` 或 `"00700,00939"` 或 `stocks.txt` |
| `-d, --days` | 回溯天数 | `-d 7` (最近7天) |
| `--date-range` | 日期范围 | `--date-range 2025-09-10 2025-09-17` |

注意：`-d` 和 `--date-range` 二选一。

### 可选参数

| 参数 | 默认值 | 说明 | 示例 |
|------|--------|------|------|
| `-c, --config` | `config.yaml` | 配置文件路径 | `-c custom_config.yaml` |
| `-b, --batch-size` | `5` | 批处理大小 | `-b 10` |
| `--no-filter` | 关闭 | 禁用公告过滤 | `--no-filter` |
| `--dry-run` | 关闭 | 试运行模式 | `--dry-run` |
| `--output` | 无 | 结果输出文件 | `--output result.json` |

## 🎯 使用场景和示例

### 场景1: 新增股票的历史数据补充


## 🚨 注意事项

### 性能考虑

1. **批处理大小**: 
   - 小批次 (2-5): 适用于网络不稳定或资源有限的环境
   - 大批次 (10-20): 适用于高性能环境，但可能增加内存使用

2. **日期范围**: 
   - 长时间范围会增加处理时间
   - 建议分段处理超过90天的历史数据

3. **股票数量**:
   - 大量股票建议使用文件方式管理
   - 可以分批处理减少单次运行时间

### 网络和稳定性

1. **API限制**: 脚本内置了API调用延迟，避免过度请求
2. **错误恢复**: 自动处理网络错误和重试
3. **中断处理**: 支持 Ctrl+C 优雅中断

### 数据一致性

1. **去重机制**: 自动避免重复下载已存在的文件
2. **原子性**: 每个股票的处理具有原子性
3. **状态追踪**: 完整的处理日志和状态记录
