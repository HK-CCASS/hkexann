# 执行报告：修复 PDF 落盘路径到 SSD

**时间**：2026-03-04
**状态**：✅ 已修复

---

## 问题描述

PDFs 未落到 `/Volumes/SSD/DockerData/hkexann_data`，而是存到了本地项目 `hkexann/` 目录。

## 根因分析

调用链：

```
manual_historical_backfill.py
  → historical_config = self.config.get('manual_historical_processing', {})
  → historical_config 仅含 common_keywords / announcement_categories
  → CorrectedHistoricalProcessor(config=historical_config)

corrected_historical_processor.py
  → downloader_config['downloader_integration']['download_directory']
     = config.get('downloader_integration', {}).get('download_directory', 'hkexann')
  → 因 config 无 'downloader_integration' 键，返回默认值 'hkexann'
```

`config.yaml` 中有完整的 `downloader_integration` 节（含 SSD 路径），但 `manual_historical_backfill.py` 构建 `historical_config` 时未将其传递给 `CorrectedHistoricalProcessor`。

## 修改记录

| 文件 | 行号 | 改动 |
|------|------|------|
| `manual_historical_backfill.py` | 230-234 | 在 `historical_config.update()` 中新增 `'downloader_integration': self.config.get('downloader_integration', {})` |

## 数据流修正后

```
config.yaml.downloader_integration.download_directory
  = "/Volumes/SSD/DockerData/hkexann_data"
  → historical_config['downloader_integration']['download_directory']
  → CorrectedHistoricalProcessor.config['downloader_integration']['download_directory']
  → RealtimeDownloaderWrapper 使用 SSD 路径
  → PDF 落盘至 /Volumes/SSD/DockerData/hkexann_data/{类别}/{股票代码}/
```
