## 🏗️ 系统架构概览

### 三层架构设计

1. **传统架构** (Legacy)
   - 基于`main.py`的单体架构
   - 代码量：2,100+行
   - 适用场景：简单任务、向后兼容

2. **微服务架构** (Microservices)
   - 分布式微服务设计
   - 代码量：29,000+行
   - 适用场景：生产环境、高并发

3. **现代统一架构** (Modern)
   - 企业级模块化设计
   - 支持策略模式和依赖注入
   - 适用场景：新功能开发、企业级应用

### 数据库系统集成

| 数据库 | 用途 | 数据类型 | 特点 |
|--------|------|----------|------|
| **MySQL** | 股票元数据 | 结构化数据 | 关系型、事务支持 |
| **ClickHouse** | 时序公告数据 | 时序数据 | 高性能查询、列式存储 |
| **Milvus** | 向量存储 | 向量数据 | 语义搜索、相似度检索 |
| **Redis** | 缓存 | 键值数据 | 高速缓存、会话管理 |

### 外部服务集成

- **HKEX官方API**: 港交所公告数据源
- **SiliconFlow API**: AI服务（文本嵌入、LLM）

## 🔄 数据流概览

### 核心处理流程

```
HKEX API → 双重过滤 → 并发下载 → 向量化处理 → 多维存储
    ↓           ↓           ↓           ↓           ↓
  实时监听    智能分类    文件存储    语义索引    快速检索
```

### 关键特性

1. **实时监听**: 60秒轮询间隔
2. **双重过滤**: 股票过滤 + 类型过滤
3. **并发处理**: 最多5个并发下载
4. **向量化**: 4096维向量嵌入
5. **多维存储**: 文件 + 关系型 + 向量数据库

## 📊 性能指标

- **代码总量**: 29,000+ 行Python代码
- **文件数量**: 56个Python文件，20+配置文件
- **处理能力**: 日均30,000+条公告记录
- **并发度**: 最多5个并发下载任务
- **向量维度**: 4096维向量嵌入
- **批处理**: 15个文档一批向量化

## 🛠️ 技术栈

- **核心语言**: Python 3.8+
- **异步框架**: AsyncIO, aiohttp
- **数据库**: PyMySQL, clickhouse-driver, pymilvus
- **AI服务**: SiliconFlow API
- **文档处理**: PDF解析、文本分块
- **配置管理**: YAML配置、环境变量

## 📈 使用建议

### 新功能开发
- 推荐使用 **现代统一架构** (`unified_downloader/`)
- 支持企业级功能和高度可配置

### 生产部署
- 推荐使用 **微服务架构** (`services/`)
- 高可用性和可扩展性

### 简单任务
- 可使用 **传统架构** (`main.py`)
- 向后兼容、易于维护
