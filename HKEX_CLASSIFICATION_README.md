# HKEX官方分类系统集成

## 概述

本次更新集成了HKEX官方的公告分类体系，大幅提升了过滤系统的精确性和性能。通过利用HKEX API提供的`T1_CODE`和`T2_CODE`字段，实现基于官方分类代码的精确过滤。

## 🚀 主要特性

### 1. HKEX分类文件解析器
- **文件**: `services/monitor/classification_parser.py`
- **功能**: 解析`HKEX--分类信息-*.csv`文件，建立完整的分类映射
- **特性**:
  - 支持227个3级分类的精确映射
  - 提供分类代码到名称的双向查找
  - 智能的排除类别转换

### 2. 增强的双重过滤器
- **文件**: `services/monitor/dual_filter.py`
- **新功能**:
  - 支持基于`T1_CODE`/`T2_CODE`的精确过滤
  - 自动将排除类别名称转换为分类代码
  - 降级策略：分类代码 → LONG_TEXT → 智能分类
  - 100%向后兼容

### 3. 配置增强
- **文件**: `config.yaml`
- **新增配置**:
  ```yaml
  dual_filter:
    hkex_classification_enabled: true  # 启用HKEX分类系统
  ```

## 📊 技术优势

### 精确性提升
- **传统方式**: 基于文本匹配，容易出错
- **新方式**: 基于数字代码匹配，无歧义
- **示例**: `"翌日披露報表"` → `"50100"` (精确匹配)

### 性能优化
- **字符串比较**: O(n*m) 复杂度
- **数字比较**: O(1) 复杂度
- **预期提升**: 10-100倍性能改善

### 维护性改善
- **分类代码**: 相对稳定，维护成本低
- **自动转换**: 配置仍然使用直观的类别名称
- **智能降级**: 确保系统鲁棒性

## 🔧 使用方法

### 1. 启用HKEX分类系统
```yaml
# config.yaml
dual_filter:
  hkex_classification_enabled: true  # 默认启用
  excluded_categories:
    - "翌日披露報表"  # 仍然使用名称，自动转换为代码
```

### 2. 测试集成
```bash
# 运行完整测试
python3 test_hkex_classification.py

# 运行系统测试模式
python3 start_enhanced_monitor.py -t
```

### 3. 查看统计信息
系统会自动显示HKEX分类系统的统计信息：
```
✅ HKEX分类系统已启用，加载了 227 个分类
📊 公告类型分布: {'23400': 1, '50100': 1}
📈 过滤统计: 保留 1 条，过滤 1 条
```
